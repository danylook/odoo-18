3PL Portal — CSV Import Guide

Step-by-step instructions for bulk importing Products, Inbound Orders, and Outbound Orders

Version 1.0 — May 2026
🖨 Print / Save as PDF

1. Overview

The 3PL portal supports bulk import via CSV files for three data types:

  • Products — add new products to your catalogue
  • Inbound Orders — create warehouse receiving operations in bulk
  • Outbound Orders — create delivery/dispatch operations in bulk

All three imports are accessed from the Import section in the sidebar, or from the dashboard Bulk Import tile grid.

Format: All files must be CSV (comma-separated values) with UTF-8 encoding. The first row must be the header row with exact column names as documented below. Column names are case-insensitive.

2. How to Create a CSV File

Using Microsoft Excel / LibreOffice Calc

  1. Open Excel or LibreOffice Calc.
  2. Enter the column headers in row 1 (exact names from this guide).
  3. Fill in your data from row 2 onward.
  4. Click File → Save As and choose CSV (Comma delimited).
  5. Encoding should be UTF-8. In Excel choose "CSV UTF-8 (Comma delimited)".
Tip: Download the sample file from each import page. It already has the correct headers — just replace the example rows with your real data.

3. Import Products

Use this import to create new storable products in your 3PL catalogue. Each row creates one product. Products will be assigned to your account automatically.

Import URL: /my/3pl/import/products

Column Reference

Column NameRequiredTypeDescriptionExample
name Required Text Product display name. Must be unique in the system. Widget A Blue
internal_reference Optional Text Internal SKU or reference code. Used when creating orders by reference. WGT-001-BLU
sales_price Optional Decimal Unit sales price. Use a dot as decimal separator (e.g. 12.50). 12.50
weight Optional Decimal Unit weight in kilograms. Used for shipping cost calculations. 0.75
barcode Optional Text EAN-13, UPC, or any barcode number. Must be unique if provided. 1234567890128

Sample File

name,internal_reference,sales_price,weight,barcode
Widget A Blue,WGT-001-BLU,12.50,0.75,1234567890128
Box Medium,BOX-MED,3.00,1.20,
Pallet Wrap,PLT-WRAP,8.99,0.30,9876543210012
Note: If a product with the same name already exists, a duplicate will be created with the same name. Check your catalogue before importing to avoid duplicates.

4. Import Inbound Orders

Use this import to create bulk receiving (inbound) operations. Each unique order_ref creates one warehouse receipt. Multiple rows with the same order_ref are combined into a single order with multiple product lines.

Import URL: /my/3pl/import/inbound

Column Reference

Column NameRequiredTypeDescriptionExample
order_ref Required Text Your purchase order number or reference. Rows with the same order_ref are grouped into one picking. PO-2026-001
product_ref One of * Text Internal reference (SKU) of the product. Takes priority over product_name if both are provided. WGT-001-BLU
product_name One of * Text Product name (partial match). Used as fallback when product_ref is empty or not found. Widget A Blue
quantity Required Decimal Number of units to receive. Must be greater than 0. 100
scheduled_date Optional Date Expected arrival date in YYYY-MM-DD format. Applied to all lines of the same order_ref. 2026-05-20
note Optional Text Special handling instructions. Applied at order level. Handle with care — fragile

* Either product_ref or product_name must be provided per row. Rows where neither resolves to a known product are skipped with an error message.

Sample File

order_ref,product_ref,product_name,quantity,scheduled_date,note
PO-2026-001,WGT-001-BLU,,100,2026-05-20,Handle with care
PO-2026-001,BOX-MED,,50,2026-05-20,
PO-2026-002,,Pallet Wrap,200,2026-05-21,
PO-2026-003,PLT-WRAP,,75,2026-05-22,Urgent delivery
Result: The sample above creates 3 inbound orders: PO-2026-001 with 2 product lines (100 × Widget + 50 × Box), PO-2026-002 with 1 line (200 × Pallet Wrap), and PO-2026-003 with 1 line (75 × Pallet Wrap).

5. Import Outbound Orders

Use this import to create bulk delivery (outbound) operations. The same grouping rule applies: rows with the same order_ref become one outbound picking with multiple lines.

Import URL: /my/3pl/import/outbound

Column Reference

Column NameRequiredTypeDescriptionExample
order_ref Required Text Your sales order number or reference. Rows sharing an order_ref are grouped. SO-2026-001
product_ref One of * Text Internal reference (SKU) of the product. WGT-001-BLU
product_name One of * Text Product name (partial match fallback). Widget A Blue
quantity Required Decimal Number of units to dispatch. Must be greater than 0. 30
scheduled_date Optional Date Target dispatch date in YYYY-MM-DD format. 2026-05-22
delivery_address Optional Text Full delivery address as plain text. Stored in the order notes. Applies to the whole order (first row wins). 123 Main St, Springfield, 12345
note Optional Text Special delivery instructions. Fragile — do not stack

* Either product_ref or product_name must be provided per row.

Sample File

order_ref,product_ref,product_name,quantity,scheduled_date,delivery_address,note
SO-2026-001,WGT-001-BLU,,30,2026-05-22,123 Main St Springfield 12345,
SO-2026-001,BOX-MED,,10,2026-05-22,,
SO-2026-002,,Pallet Wrap,5,2026-05-23,456 Oak Ave Town 67890,Fragile

6. Common Rules & Tips

File encoding

Always save as UTF-8. If you use special characters (accents, ñ, etc.) and the file is saved in Windows-1252 or Latin-1, those characters will appear garbled. In Excel use File → Save As → CSV UTF-8 (Comma delimited).

Decimal numbers

Use a dot as the decimal separator (e.g. 12.50 not 12,50). Comma-separated numbers will cause an import error on that row.

Dates

Use YYYY-MM-DD format (e.g. 2026-05-20). Other formats are not guaranteed to parse correctly.

Empty columns

Leave optional cells blank — do not fill them with dashes or N/A. An empty cell is safe; a non-numeric value in a numeric field will cause an error on that row.

Error handling

The importer processes all rows. Rows with errors are skipped and listed in the error summary shown after upload. Valid rows are imported even when some rows fail. You can fix the problem rows and re-import just those rows.

Duplicate orders

There is no duplicate-detection for orders. If you import the same file twice, you will get duplicate pickings. Always verify the result in My Orders after import.

Product must exist

For order imports, the product must already exist in the system (either created manually or imported via the Products import). If a product cannot be found the row is skipped.

7. Recommended Workflow

  1. Import Products first — ensure all products exist with their internal references (SKU codes) before importing orders.
  2. Download the sample CSV for the import type you need and use it as a template.
  3. Fill in your data in the spreadsheet, then save as CSV UTF-8.
  4. Upload on the import page and review the success/error message.
  5. Check My Orders (Inbound or Outbound) to confirm the created pickings look correct.
  6. Contact your 3PL account manager to confirm scheduling and any special requirements.
Need help? Contact our operations team or refer to the inline format table shown on each import page. You can also download the sample CSV directly from those pages.
